或许是 Apple 最没有存在感的服务：macOS Server 消亡史

2022 年 4 月 21 日，Apple 公司在一份 支持文档 中正式停止了对 macOS 的服务器软件包 macOS Server 的支持。事实上，自 2017 年发布的 macOS High Sierra 开始，macOS Server 所包含的主要功能就已经被整合到了操作系统之中，经过了将近五年的不断淘汰，这项服务终于比预想更晚地走到了生命历程的尽头。

自 2022 年 4 月 21 日起，Apple 已停止使用 macOS Server。现有的 macOS Server 用户可以继续下载该应用程序并将其与 macOS Monterey 一起使用。

后乔布斯时代，符合大众认知的 Apple 公司始终是一家面向消费者的硅谷企业，如今这家公司设计和销售的无一不是针对不同用户需求进行高度整合的产品，而非专供企业和机构的一揽子解决方案。尽管大多数用户对 macOS Server 的态度无外乎「很抱歉以这种方式认识你」，但这项几乎没什么存在感的服务的历史却要一直追溯到 1993 年——如果说每个成功的商业公司都会在其漫长的发展历程中经历一段十分艰难的黑暗时期，那么 90 年代无疑是 Apple 公司历史上的「中世纪」。

自乔布斯从 1983 年开始在公司内部受到排挤直至 1985 年无奈出走，Apple 公司的产品线也随着其掌控力的弱化而变得混乱不堪，接替乔布斯主管开发工作的是法国人让 - 路易 · 加西（Jean-Louise Gassee），这位地道的老巴黎想要把 Apple 公司打造成硅谷的香奈儿，一上台就叫停了所有定位中低端的项目，希望以此拉高产品的单件利润。加西的想法得到了时任 CEO 约翰 · 斯卡利（John Sculley）的支持，得益于 Apple 公司的品牌号召力，Apple 产品的售价在两位大聪明的通力合作之下与日俱增，水涨船高的销售额也给了他们继续推行这一战略的信心，而丝毫意识不到这种不正常的增长实质上是在消费公司长期以来积攒的用户口碑。

斯卡利的硅谷香奈儿从 1985 年一直香到了 90 年代，然后就香不下去了。1990 年开始，Apple 产品的销量开始不及预期，并且在此后逐年下滑。尽管 IBM PC 及其兼容机的崛起是主要原因，但加西让 Apple 产品从定位到售价全数脱离普通用户的做法的确起到了推波助澜的作用。在财报上坐了一趟过山车的斯卡利认真反思了自己在决策上的重大失误，而反思的结果就是让法国人走出去，把德国人引进来——Apple 欧洲 CEO 迈克尔 · 斯宾德勒（Michael Spindler）成为了新的首席运营官，试图依靠精准的德国工艺将 Apple 公司点歪了的科技树拉回正轨。

虽然此前的战略被证明是竭泽而渔，但尝到了甜头的斯卡利并不想放弃即将到手的蛋糕。割普通消费者的韭菜显然不是长久之计，1993 年 3 月 22 日，Apple 公司正式推出了服务器产品线 Workgroup Server，与长期占据企业市场的 IBM 展开了正面竞争。可惜的是斯卡利并没能盼来胜利的曙光，他于同年 10 月 15 日被董事会罢免，接替他的则是斯宾德勒。

即使不与 IBM 作对比，Apple 公司的 Workgroup Server 也并不成功。除了 1996 年 2 月的 Workgroup Server 9150 之外，其余所有的 Workgroup Server 服务器都是基于同时期 Macintosh 电脑修改而来。也许正是因为这个原因，斯宾德勒及其继任者保留了 Workgroup Server，直到乔布斯在 1997 年如闪电般归来，Apple 公司才在大刀阔斧的产品线精简过程中砍掉了这一项目——然后将其换成了更进一步的机架式服务器 Xserve。

早期的 Xserve 本质上依然是一台 PowerPC 处理器驱动的电脑，只是将预装的系统换成了专供服务器的 Mac OS X Server 10.0。这一操作系统最早于 1999 年 3 月 16 日发布第一版，前身则是乔布斯在出走期间创建的 NeXT 公司开发的 NeXTSTEP 平台。Apple 公司通过收购 NeXT 公司的方式迎回了乔布斯，而 NeXTSTEP 也就理所应当的取代了老旧的 Mac OS 9（与之前的版本统称为 Classic Mac OS），进而成为了 Mac OS X（2016 年更名为 macOS，至今）乃至 iOS、iPadOS、watchOS 和 tvOS 的基础。

最初的 Mac OS X Server 1.0 是一个融合了 Mac OS 8.5、NeXTSTEP 平台以及各种开源组件，带有强烈试验性质的产物，旨在为正在开发中的正式商业版本开路。Mac OS X Server 1.0 的构成既包括面向消费者的 Mac OS X，也包括面向企业的 Mac OS X Server，二者先后于 2001 年 3 月 24 日和 5 月 21 日发售，起始版本号跳到了 10.0，与命名中的罗马数字 X 相对应。从此之后的十年间，Mac OS X Server 一直保持着对 Mac OS X 亦步亦趋的更新节奏，直到 2011 年 1 月 31 日最后一代 Xserve Xeon 服务器正式停产，Apple 公司才停止了对 Mac OS X Server 的支持。

Xserve + Mac OS X Server 的组合注定无法与 IBM 提供的解决方案相抗衡，其中的一个主要原因就是 Apple 公司不肯放松对自家产品的控制权，而是希望企业也能像使用普通的 Mac OS X 一样「优雅」地完成管理工作。然而 Mac OS Server 闭源搭配开源组件的形式难以满足企业对服务器功能的定制需求，软硬件深度捆绑的机制强行拉高了维护成本和升级难度，而这些代价换来的仅仅是一个好看的图形化用户界面，的确有些得不偿失。

企业不买账，Apple 公司被迫将目光转向了办公室和学校这些体量较小、更易于部署产品的服务对象。由于 Mac OS X Server 本质上是 Mac OS X 加上专用的服务器组件，因此 Apple 公司也改变了自身的战略：自 Mac OS X 10.7 开始，Mac OS X Server 将作为软件包而非独立的分支操作系统出售，任何用户都可以在符合要求的 Mac 电脑上安装这一软件包，从而将其作为服务器使用；而 Mac OS X Server 的功能重心也随之发生偏移，Apple 希望以此在办公室和学校打造由 Mac 电脑构建的办公和教育环境，管理者无需部署额外的服务器，就可以在自己的 Mac 电脑上利用现有的网络直接管理组织成员的设备。

软件包形式的 Mac OS X Server 最重要的功能是描述文件管理器，这一功能允许管理者通过创建一个包含邮件、日历、通讯录和信息的用户帐户、系统设置、实行限制以及 PIN 和密码规则等信息的描述文件，组织内部的其他成员只需安装描述文件就可以完成设备的部署，从而与管理者保持同步。有趣的是，描述文件也允许管理者无需使用 Apple ID 就可以直接推送通过 App Store 批量购买计划所购得的机构授权 app 到指定的设备上，前些年层出不穷的各种「Apple 助手」所谓「免 Apple ID 安装应用」 的功能就是钻了这个空子。

从 2015 年 9 月 16 日发布的 OS X 10.11 开始，Mac OS X Server 不再添加任何新功能，这一版本被称为 Server 5.0，也是 Mac OS X Server 的最后一次大版本更新；2016 年 9 月 20 日，OS X 更名为 macOS，Mac OS X Server 随之更名为 macOS Server。从 2018 年 9 月 28 日的 macOS Server 5.7 开始，Apple 公司删除了 macOS Server 中除描述文件管理器、Open Directory 和 Xsan 之外的其他功能，这似乎预示了 macOS Server 的命运。

2021 年 12 月 8 日随 macOS 12 发布的 macOS Server 5.12 是最后一个版本，描述文件管理器将被更加轻量化的 MDM 解决方案 取代——Apple 为 macOS Server 的各项功能都找到了归宿，从服务器到 Mac 电脑，从独立的操作系统到作为「DLC」单独出售的软件包，macOS Server 没能如 Apple 公司之愿成长为优秀的 UNIX 分支，反而是「越长大越孤单」，连声名狼藉都没落的，只剩下无人问津。macOS Server 如今的境遇折射出了 Apple 公司作为科技行业所谓「消费品企业」的本质——所谓术业有专攻，Apple 公司可以用 iPhone 改变一切，甚至「再一次，改变一切」，而 macOS Server 就只能改变自己了。
